## State Management

### Global State (Context)
- **AuthContext**: User authentication state, login/logout functions
- **ThemeContext**: Dark/light mode preference

### Local State (useState)
- Component-specific data (devices, vulnerabilities, scans)
- UI state (loading, errors, filters)
- Form inputs

### Server State
- Fetched via services
- Cached in component state
- Updated via WebSocket

## Security

### JWT Token Flow
1. User logs in
2. Backend returns JWT token
3. Token stored in localStorage
4. Axios interceptor adds token to all requests
5. Backend validates token
6. If 401, redirect to login

## Folder Structure Philosophy

```
src/
в”ңв”Җв”Җ components/     вҶ’ Reusable UI components (presentational)
в”ңв”Җв”Җ pages/          вҶ’ Full page components (container)
в”ңв”Җв”Җ services/       вҶ’ API communication layer
в”ңв”Җв”Җ hooks/          вҶ’ Custom React hooks (business logic)
в”ңв”Җв”Җ context/        вҶ’ Global state management
в””в”Җв”Җ utils/          вҶ’ Helper functions (pure functions)
```

## Design Patterns Used

1. **Container/Presentational Pattern**: Pages (containers) use Components (presentational)
2. **Custom Hooks Pattern**: Reusable logic (useAuth, useDevices, useWebSocket)
3. **Service Layer Pattern**: Separate API calls from components
4. **Context Pattern**: Global state without prop drilling
5. **Protected Route Pattern**: Authentication wrapper

---

**This architecture ensures:**
- вң… Separation of concerns
- вң… Reusability
- вң… Maintainability
- вң… Scalability
- вң… Testability
